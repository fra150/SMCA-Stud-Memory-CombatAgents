## рҹ§  What is BAS?

Current LLMs fail at long context. **BAS changes the paradigm.**

Instead of asking one model to read 500 pages (and inevitably forget), BAS creates **500 specialized agents**, each expert on one page, connected by StudSar as a unified brain. It is a **dynamic cognitive architecture** that combines:

- **Dynamic agent scaling** вҖ” N agents = N document segments (not fixed)
- **Semantic selection** вҖ” agents chosen by cosine similarity, not position
- **Post-retrieval reasoning** вҖ” dedicated executor for numerical aggregation
- **Context-aware filtering** вҖ” pre-calculation filtering by transaction type
- **Perfect needle retrieval** вҖ” 100% accuracy on facts hidden in 50-segment documents

At its core lives **StudSar Centrale** вҖ” the central memory system that transforms static document segments into living associative knowledge, enriched with emotional tags and temporal metadata.

---

## рҹҸӣпёҸ Architecture вҖ” The Five Pillars

### `01` вҖ” BAS Engine В· *The Neural Cortex*

Dynamic agent creation where document complexity dictates agent count. Runs entirely on standard CPU using `sentence-transformers` for embedding generation. Each agent holds exactly one segment, creating true expertise rather than generalized forgetting.

### `02` вҖ” Semantic Selection В· *The Thalamus*

No positional bias. The system calculates cosine similarity between query embedding and segment embeddings, selecting only the most relevant agents for combat. **Impact:** Needle accuracy improved from 50% вҶ’ 100%.

### `03` вҖ” Post-Retrieval Executor В· *The Prefrontal Cortex*

A dedicated numerical reasoning layer that separates retrieval from computation. Features multi-language keyword mapping (Italian "entrate" вҶ’ English "income"), strict type filtering, and automatic net balance computation (income - expense).

### `04` вҖ” StudSar Centrale В· *The Hippocampus*

The shared memory system connecting all agents as a unified brain. Agents don't just retrieve text вҖ” they access living markers with emotional resonance, usage frequency, and temporal decay. Memory coherence monitoring via TMDR (Temporal Memory Decay Regression).

### `05` вҖ” Giudice BAS В· *The Executive Control*

Emergent standards evaluation with confidence scoring. The Judge determines when aggregation requires the Post-Retrieval Executor versus direct answer generation. When uncertainty exceeds thresholds, **Dio** (the User) intervenes.

## рҹҡҖ Quick Start

### Prerequisites

```bash
# Activate your virtual environment and install dependencies
# Windows PowerShell
.venv\Scripts\Activate.ps1
pip install -r requirements.txt

# Linux / macOS
source .venv/bin/activate
pip install -r requirements.txt
```

### Run the BAS Demo

End-to-end orchestration: loads financial documents, creates dynamic agents per segment, executes semantic selection, runs post-retrieval numerical reasoning, and outputs the Risposta Suprema with confidence scores.

```bash
python bas_demo.py
```

> Benchmark results are saved to `/locomo_benchmark_results.json` вҖ” visualizations and state land in `--output-dir` (default: `bas_output/`)

## рҹ”§ Usage Examples

```bash
# Single document demo with dynamic agent scaling
python bas_demo.py \
  --document "financial_report_50seg.txt" \
  --query "What is the total revenue?" \
  --output-dir bas_results
```

```bash
# Run full LOCOMO benchmark suite (30 tests)
python bas_demo.py \
  --run-benchmark \
  --benchmark-tests 30 \
  --segment-sizes 20,50 \
  --output-dir locomo_results
```

```bash
# Compare BAS vs SMCA on same document
python bas_demo.py \
  --compare-smcas \
  --document "test_doc.txt" \
  --query "Calculate net budget balance" \
  --agents-bas dynamic \
  --agents-smca 10
```

```bash
# Post-retrieval executor with strict context filtering
python bas_demo.py \
  --document "mixed_transactions.txt" \
  --query "Entrate totali" \
  --filter-type income \
  --calculate-net \
  --language it
```
